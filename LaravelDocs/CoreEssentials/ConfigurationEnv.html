配置与环境 ​

一、配置文件 ​

是什么？

  • Laravel的config目录下存放的PHP文件，用来集中管理项目设置。

有什么用？

  • 统一存放数据库连接、邮件设置等配置项，修改配置不用到处找代码。

怎么用？

  • 1.查看配置：config/app.php 是核心配置文件
  • 2.获取配置值：
php
// 获取应用时区
$timezone = config('app.timezone');

// 获取数据库配置
$dbHost = config('database.connections.mysql.host');

  • 3.修改配置：
php
// 临时修改时区（仅当前请求有效）
config(['app.timezone' => 'Asia/Shanghai']);
1
2

常用配置文件

文件用途重要配置项
app.php核心设置时区、语言、加密密钥
database.php数据库连接MySQL、Redis配置
mail.php邮件设置SMTP服务器、发件人
cache.php缓存配置Redis、Memcached驱动
filesystems.php文件存储本地存储、云存储

二、.env使用 ​

是什么？

  • 项目根目录下的环境变量文件（.env），像项目的"密码本"。

有什么用？

  • 安全存储敏感信息（数据库密码、API密钥），不同环境用不同配置。

怎么用？

  • 1.基础格式：
env
KEY=VALUE
# 注释以#开头

APP_NAME=Laravel
DB_PASSWORD=my_secret_password

三、环境访问 ​

是什么？

  • 获取环境变量的方法，区分开发、生产等环境。

有什么用？

  • 根据运行环境自动切换配置，比如：
    • 开发环境：显示详细错误
    • 生产环境：隐藏错误信息

怎么用？

  • 1.检测当前环境：
php
// 检查是否是本地环境
if (app()->environment('local')) {
    // 开发环境特殊逻辑
}

// 检查是否是生产环境
if (app()->environment('production')) {
    // 生产环境特殊逻辑
}

最佳实践 ​

1.安全第一：

  • 敏感信息只放.env
  • 生产环境设置 APP_DEBUG=false

2.多环境配置： 创建不同环境的文件：

bash

常见问题 ​

❓ Q：修改了.env为什么没生效？
→ 需要重启服务：php artisan serve 或清除缓存：php artisan config:clear

❓ Q：如何在代码中安全使用API密钥？
→ 永远不要硬编码：

php
// 错误做法 ❌
$apiKey = "sk_live_123456";

// 正确做法 ✅
$apiKey = env('STRIPE_API_KEY');
1
2
3
4
5

❓ Q：团队协作时如何同步配置？

1.提交.env.example到Git
2.新成员复制后重命名：

bash
cp .env.example .env
1

3.生成应用密钥：

bash
php artisan key:generate
1

❓ Q：生产环境忘记设置APP_DEBUG怎么办？
→ 在AppServiceProvider中强制设置：

php
public function boot()
{
    // 强制生产环境关闭调试
    if ($this->app->environment('production')) {
        config(['app.debug' => false]);
    }
}
