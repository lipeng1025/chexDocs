nuxt.config.ts ​

核心概念 ​

nuxt.config.ts 是 Nuxt3 项目的核心配置文件，控制框架行为、模块集成、构建设置和应用元数据。

1. 基本结构 ​

typescript
import { defineNuxtConfig } from 'nuxt'

export default defineNuxtConfig({
  // 所有配置项写在这里
})

控制全局应用行为和头部设置

3.常见错误

  • 模块顺序问题：某些模块依赖顺序（如 Pinia 需在 i18n 之前）
  • 路径别名冲突：避免自定义别名与 Nuxt 默认冲突
  • 环境变量未加载：确保 .env 文件存在且变量前缀正确
  • 类型错误：运行 npx nuxi typecheck 检查

💡 核心提示：nuxt.config.ts 是 Nuxt3 项目的控制中心，建议：

  1. 按功能模块分区配置
  2. 使用 JSDoc 注释说明重要配置
  3. 定期检查官方配置变更
  4. 复杂项目拆分为多个配置文件
typescript
import apiConfig from './config/api'  
import uiConfig from './config/ui'  

export default defineNuxtConfig({  
  ...apiConfig,  
  ...uiConfig  
})

// nuxt.config.ts
export default defineNuxtConfig({
  // 1. 应用配置
  app: {

    pageTransition: {},

  // 5. 目录配置

    assets: 'assets',
    layouts: 'layouts',
    middleware: 'middleware',
    pages: 'pages',
    public: 'public',
    plugins: 'plugins'
  },
  
  // 6. 自动导入配置
  imports: {
    dirs: [],
    global: false
  },
  
  // 7. 别名配置
  alias: {},
  
  // 8. 构建配置
  build: {},

  vite: {},
  
  // 10. Webpack 配置（不推荐，优先使用 Vite）
  webpack: {},
  
  // 11. CSS 配置
  css: [],
  
  // 12. 开发工具配置

    enabled: false
  },
  
  // 13. 实验性功能
  experimental: {},
  
  // 14. TypeScript 配置

    strict: true,
    typeCheck: true
  },
  
  // 15. SSR 配置
  ssr: true,
  
  // 16. Nitro 配置
  nitro: {
    routeRules: {},

    devProxy: {}
  },
  
  // 17. 路由配置
  router: {},
  
  // 18. 钩子配置
  hooks: {},

  postcss: {},
  
  // 20. 源文件配置

  // 21. 调试配置

  features: {},
